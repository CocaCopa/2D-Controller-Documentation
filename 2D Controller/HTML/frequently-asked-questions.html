Frequently Asked Questions

Q: How can I set up a new character?

  • A:
  • To set up a new character, please consult the Locomotion System page in the documentation. By following the outlined steps, you can create a fully functional base character to kickstart your project.

Q: Why does the character need 2 colliders in order to function properly?

  • A:
  • The character requires two colliders for optimal functionality, primarily due to the mechanics implemented in the Locomotion System. One collider (Vertical) is dedicated to the character's main body collision detection, while the second collider (Horizontal) is specifically utilized for the floor slide mechanic. The system dynamically toggles between these two colliders as needed, ensuring that each one is active only when required. This dynamic switching allows for precise control over collision detection, enabling the character to seamlessly interact with the environment.
    For more insight, explore the documentation accompanying CharacterEnvironmentalQuery.SetActiveCollider, which offers more details on the usage and functionality.

Q: How do I begin implementing the provided systems via code?

  • A:
  • To begin, create a new controller class inheriting from HumanoidController. This grants access to all implemented locomotion mechanics, depending on which optional components are attached to your character object. For combat, ensure your character object also has the CharacterCombat component attached. Consult the documentation for each of the other provided classes to identify public properties for creating features like UI-tracked cooldowns or integrating custom code.

Q: How can I make a charge attack release immediately upon key release, bypassing the need for a draw animation cycle?

  • A:
  • By default, the code for executing a charged attack verifies whether the animation cycle is finished before permitting the attack to be released. However, if you're utilizing solely a charging animation without a corresponding draw animation, this verification process will cause a delay in releasing the attack, which might be an unwanted behaviour in this context. To bypass this delay, you can create a draw animation consisting of only one frame. This way, the animation check completes immediately, and upon key release, the attack will be promptly executed.
