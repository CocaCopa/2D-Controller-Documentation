CharacterEnvironmentalQuery

SWITCH TO MANUAL

Description

The CharacterEnvironmentalQuery class serves as the central hub for managing collision checks within this asset. Some of the scripts in this asset, seeking specific details about collisions within the game environment will utilize this class.

By consolidating collision check functions into a single class, the asset's modularity and maintainability are enhanced. Encapsulating collision logic within this class not only reduces code duplication but also improves the overall organization of the project.

To incorporate this class into your project, attach it to any gameObject intended to represent a humanoid character. Once attached, you can reference it in your scripts to gain access to its public methods. Ensure that the class is attached to the main gameObject in question, as it utilizes the character's position in various scenarios.


Important Notes

All of the method examples assume that the CharacterEnvironmentalQuery class, as well as each of the Example classes are attached to the same gameObject.

Each method below includes default cast distances for the physics collision queries based on the character's collider. Additionally, the ledge grab, head, chest and feet collision queries require the provision of a transform.


Public Methods
SetActiveCollider Sets the collider to be utilized by the class.
ColliderAroundCharacter Searches for colliders around the character in the specified range.
GroundCheck Checks if the character steps on a platform.
WallAbove Checks if there is a wall above the character's head.
WallInFront Checks if there is a wall in front of the character.
LedgeGrabCheck Checks if the character is able to grab onto a ledge.
HeadCollisionCheck Checks if there is a collider in front of the character's head.
ChestCollisionCheck Checks if there is a collider in front of the character's chest.
FeetCollisionCheck Checks if there is a collider in front of the character's feet.
