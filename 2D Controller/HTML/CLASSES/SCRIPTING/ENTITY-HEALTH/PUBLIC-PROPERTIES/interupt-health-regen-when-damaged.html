EntityHealth.InteruptHealthRegenWhenDamaged

Declaration

public bool InteruptHealthRegenWhenDamaged { get; set; }


Description

Indicates whether your entity should stop regenerating health, when they take damage.

This property controls whether the entity's ongoing health regeneration process is interrupted upon sustaining damage during gameplay. When set to true, any ongoing health regeneration will cease immediately upon the entity taking damage, preventing the simultaneous recovery of health and damage. Conversely, when set to false, the entity's health regeneration continues uninterrupted even when damage is sustained, allowing for potential simultaneous healing and damage-taking.


In this example, the character will be able to regenerate health without interruption only when their current health exceeds 50%. The following script should be attached to your character. 

using UnityEngine;

[RequireComponent(typeof(EntityHealth))]
public class Example : MonoBehaviour
{
    private EntityHealth healthScript;

    private void Awake()
    {
        healthScript = GetComponent<EntityHealth>();
    }

    private void Update()
    {
        float currentHealth = healthScript.CurrentHealthPoints;
        float maxHealth = healthScript.MaxHealthPoints;

        if (currentHealth / maxHealth > 0.5f)
        {
            healthScript.InteruptHealthRegenWhenDamaged = true;
        }
        else
        {
            healthScript.InteruptHealthRegenWhenDamaged = false;
        }
    }
}
