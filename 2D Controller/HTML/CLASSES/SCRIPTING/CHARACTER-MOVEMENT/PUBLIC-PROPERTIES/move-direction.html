CharacterMovement.MoveDirection

Declaration

public float MoveDirection { get; set; }


Description

This property represents the direction your character is facing.

It is important to note that the CharacterMovement class operates independently of external changes in your character's direction. In the event of a sudden shift in movement direction, it is crucial to manually adjust the MoveDirection property accordingly.


Unfortunately, providing a specific example demonstrating when and how to update the value of MoveDirection is not feasible here, as it requires a custom action to precede it. Instead of presenting an example in this context, please refer to the CharacterDash_OnDashDistanceCovered event listener in the HumanoidController script. Navigate to your project files:
Assets -> 2D Character Controller -> _Scripts -> Character -> Controllers -> HumanoidController.cs

When the character executes a dash, the MoveDirection property is updated with the value of the character's right direction (transform.right). This is done because it is possible for your character to jump, running toward the left direction, but the dash could be performed in the right direction and vice versa. This would result to your character travelling toward the right direction as expected, but as soon as the dash was completed, they would change their travel direction again, to the left direction.
Updating the value of the MoveDirection property accordingly, solves this issue.
