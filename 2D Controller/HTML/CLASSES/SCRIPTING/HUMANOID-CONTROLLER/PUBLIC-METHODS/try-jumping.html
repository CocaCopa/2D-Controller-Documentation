HumanoidController.TryJumping

Declaration

protected void TryJumping();


Description

Attempts to make the character jump based on certain conditions.

Responsible for initiating a jump action for the character based on specific conditions. Upon invocation, the method evaluates predefined conditions, such as the character's grounded state or any imposed jump restrictions. If the conditions are satisfied, the character is prompted to execute a jump, ensuring adherence to established gameplay mechanics.
Moreover, triggering a jump will override any ongoing attack animation within the combat system. To mitigate such occurrences or implement other custom restrictions, manual customization within your script is necessary. Below, an example scenario is provided to illustrate potential customizations.


Here's an example of calling the function to prompt your character to attempt a jump, utilizing the default behavior checks: 

using UnityEngine;

public class Example : HumanoidController
{
    protected override void Update()
    {
        base.Update();

        if (Input.GetKeyDown(KeyCode.Space))
        {
            TryJumping();
        }
    }
}

If you wish to tailor the jump behavior for additional restrictions, as mentioned above, manual customization is required. Here's an example of adjusting the function call to ensure it does not cancel any ongoing attacks: (Note that 'characterCombat' is a protected property of the HumanoidController, enabling its use without the need to reference it explicitly in your script.) 

using UnityEngine;

public class Example : HumanoidController
{
    protected override void Update()
    {
        base.Update();

        if (Input.GetKeyDown(KeyCode.Space))
        {
            if (!characterCombat.IsAttacking)
            {
                TryJumping();
            }
        }
    }
}
